# Introduction

> Build tools LLMs can call like APIs, locally or in the cloud.

ATP (Agent Tool Protocol) is a developer-first standard that turns your functions into tools LLMs can call via ChatATP. Think of ATP as the **SDK + runtime** for deploying capabilities to your AI assistant or agent.

It enables:

* üß† Building AI agents powered by ToolKits
* üîå A plug-and-play ToolKit architecture for LLMs
* üîê Secure integration with APIs, databases, or files
* ‚òÅÔ∏è Cloud-hosted or local deployment of tools callable by LLMs

---

## How ATP Works

ATP is the foundation of **ChatATP**, an AI system that allows any LLM to call your custom tools through a shared context protocol.

```mermaid
flowchart LR
    UserPrompt["User Prompt"] --> ChatATP["ChatATP Agent"]
    ChatATP --> ToolKit1["ToolKit A"]
    ChatATP --> ToolKit2["ToolKit B"]
    ToolKit1 -->|@register_tool| ToolA["Function A"]
    ToolKit2 -->|@register_tool| ToolB["Function B"]
```

* **ChatATP Agent**: The orchestrator that selects and routes tools to call
* **ToolKits**: Your custom packages of capabilities, created with the `atp` SDK
* **Tools**: Functions exposed with the `@register_tool` decorator
